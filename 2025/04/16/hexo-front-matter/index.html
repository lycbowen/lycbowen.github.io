Front Matter 是 Markdown 文件顶部的一段 YAML 配置。Hexo 会根据这段配置识别文章标题、发布时间、分类、标签，以及主题提供的一些扩展字段。

基本格式如下：

常用字段

  • title：文章标题，通常会显示在文章页和列表页。
  • date：发布时间，影响归档和首页排序。
  • updated：更新时间，不写时 Hexo 会根据配置自动处理。
  • tags：文章标签，适合放具体技术点。
  • categories：文章分类，适合放文章所属的大方向。

Keep 主题常用字段

Keep 主题还支持一些额外字段，例如置顶、摘要、封面图和密码访问。

其中 sticky 的数值越大，文章越靠前；excerpt 可以手动指定首页摘要；home_cover 用于设置首页文章封面。

分类和标签的建议

分类不要太碎，最好控制在几个稳定的大类里，例如：

  • Python
  • Linux
  • 工具教程
  • 机器人
  • 博客搭建

标签可以更具体，例如 Arch Linux、rsync、Fyne、ADB。这样既方便归档，也方便读者从一篇文章跳到同一主题的其他文章。

注意事项

YAML 对缩进比较敏感，建议统一使用两个空格或直接使用列表写法。字段值里如果有特殊符号，可以用引号包起来。

如果文章不想公开发布，最好不要放在 source/_posts/ 里，可以放到草稿目录或先不提交到部署分支。
